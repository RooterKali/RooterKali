# dep_hunt

`dep_hunt` is a source-map based dependency confusion scanner for authorized security research.

It identifies exposed `.js.map` files, parses `sourcesContent` with a Node AST parser, extracts `import`, `require`, and dynamic `import()` references, then scores findings as `Critical`, `High`, `Medium`, or `Low`.

## Overview

The workflow is designed for real-world assessment work:

- Crawl live targets or process pre-collected `.map` files
- Resolve sourcemap content and extract package references
- Check package status and scope exposure
- Preserve evidence in JSON and HTML reports
- Optionally notify Discord when high-priority findings appear

## Public Summary

This repository is a summary of the tool, not the full private codebase.

The private implementation stays off the public repo, while this README explains the workflow, the report style, and the kind of evidence the tool produces.

## Installation

From the project directory:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -e .
npm install
python -m playwright install chromium
```

If Playwright needs system libraries on Linux:

```bash
python -m playwright install --with-deps chromium
```

## Quick start

Scan a single domain:

```bash
dep-hunt --input example.com
```

Run a faster scan with output:

```bash
dep-hunt --input example.com --fast --output report.json
```

Scan a list of domains:

```bash
dep-hunt --input domains.txt --enum all
```

Process downloaded sourcemaps:

```bash
dep-hunt --maps ./source_maps/ --output report.json
```

Resume an interrupted run:

```bash
dep-hunt --resume example_resume.cfg
```

Read targets from stdin:

```bash
cat domains.txt | dep-hunt --fast
```

## Main Modes

- `--input` scans a single domain or a file containing domains
- `--enum` enables passive subdomain enumeration when supported tools are installed
- `--maps` processes downloaded `.map` files instead of crawling
- `--find-credentials` searches sourcemap content for exposed secrets
- `--output` writes JSON and HTML reports
- `--resume` continues an interrupted scan
- `--loop` runs the scanner on a schedule

## Output

When `--output report.json` is used, the scanner writes:

- `report.json`
- `report.html`

## Claim Helper

`claim-assist` builds a safe package scaffold from the queue of actionable candidates.

Dry run:

```bash
claim-assist --queue queue.jsonl --latest --mode dry-run
```

Publish mode:

```bash
claim-assist --queue queue.jsonl --latest --mode publish --yes
```

## Webhook Persistence

Persist a Discord webhook URL for alerts:

```bash
dep-hunt --save-webhook https://discord.com/api/webhooks/ID/TOKEN
```

Default location:

- `~/.config/dep_hunt/webhook_url`

## Project Layout

- `depi_jsmon.py` - main scanner implementation
- `dep_hunt.py` - compatibility wrapper for the console script
- `ast_parser.js` - Babel-based import extractor for sourcemap sources
- `claim_assist.py` - queue helper for actionable candidates
- `depi.py` - alternate scanner variant used during development
- `hunter.py` - experimental playground / prototype scanner
- `h1-domains_resume.cfg` - example resume state

## Notes

- This project is for authorized security testing and research only.
- Keep `node_modules/`, cache files, and generated build artifacts out of version control.
- The scoring logic is based on package status, scope exposure, and source confidence.
